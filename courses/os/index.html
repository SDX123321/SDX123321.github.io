第一章 操作系统概述

▼

1. 四个设计目标

方便性 有效性 可扩充性 开放性

2. 四个基本特征

特征含义
并发性多个程序在同一时间段内交替执行（最基本特征）
共享性系统中的资源可供多个并发执行的进程共同使用
虚拟性通过某种技术把一个物理实体变成若干个逻辑上的对应物
不确定性程序执行结果不可再现，执行顺序和时间不可预测

3. 四种结构

结构特点
整体式结构模块化设计，各模块可相互调用，结构紧凑但模块间接口复杂
层次式结构按功能分层，每层只能调用下层功能，结构清晰
C/S（客户-服务器）结构将 OS 分为若干服务器进程，客户端通过消息请求服务
微内核结构仅将最核心功能放入内核，其他功能作为用户态服务进程

4. 微内核的优势

  1. 代码量少，内核变得更简洁
  2. 充分模块化，更换任一模块不影响其他模块，方便第三方开发
  3. 当前未使用的模块不必运行，大幅减少内存需求
  4. 增强可移植性，移植时只需重点修改微内核部分，代码量很少

第二章 处理器管理

▼

1. 进程

定义：进程是具有独立功能的程序关于某个数据集合上的一次运行活动，是系统进行资源分配、调度和保护的独立单位。

三种基本状态：

运行态 就绪态 等待态（阻塞态）

四种状态转换：

转换触发条件
就绪 → 运行 CPU 调度程序选择该进程占有 CPU
运行 → 就绪 ① 时间片用完，被剥夺 CPU
② 抢占式系统中被高优先级进程抢占
运行 → 等待 ① 申请资源（如内存）但系统当前没有足够资源
② 需要 I/O 操作或等待信号/服务，但尚未准备好
等待 → 就绪 等待的资源/服务/信号可以提供，或等待的事件发生
不可能的转换：就绪 ↔ 等待（必须经过运行态）。加上挂起状态后共有 7 种状态。

进程创建原语 fork()：

返回值含义
-1创建失败
0在子进程中返回
>0在父进程中返回（返回子进程 PID）

2. 处理器调度

三级调度：

调度类型别名功能
高级调度作业调度从后备队列选择作业调入内存
中级调度平衡负载将进程在内存和外存之间交换
低级调度进程调度从就绪队列选择进程分配 CPU

周转时间相关指标：

  • 周转时间 = 完成时刻 - 到达时刻
  • 平均周转时间 = 各进程周转时间之和 / 进程数
  • 带权周转时间 = 周转时间 / 服务时间

调度算法：

算法全称特点
FCFS先来先服务非抢占式，简单公平，对长作业有利
SJF最短作业优先平均等待时间最短，但可能导致长作业饥饿
SRTF最短剩余时间优先SJF 的抢占式版本
HRRN最高响应比优先响应比 = (等待时间+服务时间)/服务时间
RR时间片轮转每个进程运行固定时间片后切换

3. 同步与互斥 — PV 操作

信号量机制：

  • P 操作（wait）：S = S - 1；若 S < 0，进程阻塞进入等待队列
  • V 操作（signal）：S = S + 1；若 S ≤ 0，唤醒等待队列中的一个进程
PV 操作要点：
• 互斥信号量初值通常为 1（mutex）
• 同步信号量初值根据资源数确定
• P 操作在"进入临界区前"，V 操作在"离开临界区后"
• 互斥的 PV 操作必须成对出现在同一进程中

经典同步问题：

  • 生产者-消费者问题：需要 3 个信号量（mutex 互斥、empty 空缓冲区数、full 满缓冲区数）
  • 读者-写者问题：读者可并发读，写者独占
  • 哲学家就餐问题：避免死锁的资源分配

4. 死锁

四个必要条件（缺一不可）：

互斥使用 不可抢占 请求与保持 循环等待

三种解决策略：

策略方法
预防（破坏条件）破坏"不可抢占"、"请求与保持"、"循环等待"之一
避免银行家算法：检查分配后系统是否处于安全状态
检测与恢复允许死锁发生，检测到后进行恢复
银行家算法核心：对每个请求，假设分配后检查是否存在安全序列。若存在则分配，否则拒绝。
安全序列：存在一个进程执行顺序，使得每个进程都能获得所需的最大资源并顺利完成。

第三章 存储管理

▼

1. 地址转换与分页管理

逻辑地址 → 物理地址：页号 → 查页表 → 页框号（物理块号）→ 物理地址

快表（TLB）查找流程：

  1. 先在快表中查找该页号（速度极快）
  2. 若命中，直接取出页框号
  3. 若未命中，访问内存中的页表，同时将该映射关系加入快表
引入快表后，有效访问时间 = 命中率 × (快表访问时间 + 内存访问时间) + (1 - 命中率) × (快表访问时间 + 2 × 内存访问时间)

2. 分段管理

  • 按程序逻辑结构分段，每段有段名和段长
  • 逻辑地址 = (段号, 段内偏移)
  • 通过段表查找：段号 → 段基址 + 段内偏移 = 物理地址
  • 段的长度可变，便于共享和保护

3. 段页式存储

结合分段和分页的优点：

  1. 先分段（逻辑划分）
  2. 每段再分页（物理管理）
  3. 地址结构：(段号, 页号, 页内偏移)
  4. 需三次访存：段表 → 页表 → 物理地址

4. 局部性原理

类型含义
时间局部性刚被访问的数据很可能在近期再次被访问（如循环变量）
空间局部性访问某个地址后，其附近的地址很可能被访问（如顺序执行）
顺序局部性程序通常按指令顺序依次执行

5. 页面置换算法

当发生缺页中断且内存已满时，需要选择一个页面换出。

算法策略特点
OPT淘汰未来最长时间内不再使用的页面理论最优，无法实现，用作参照
FIFO淘汰最早进入内存的页面简单，但可能出现 Belady 异常（增加页框数反而缺页率上升）
LRU淘汰最近最久未使用的页面性能好，接近 OPT，无 Belady 异常
Belady 异常：FIFO 算法特有现象——增加物理页框数后，缺页中断次数反而增加。LRU 和 OPT 不会出现此现象。
计算要点：
• 缺页率 = 缺页中断次数 / 总访问次数
• LRU 的实现：用栈或计数器记录各页面最后访问时间
• FIFO 的实现：用队列，最早进入的页面在队首

第四章 设备管理

▼

1. 五个设计目标

  1. 提高系统利用率，实现设备的并行运行
  2. 采用虚拟技术，实现设备的动态分配
  3. 采用缓冲技术，减小主机和外设的速度差异
  4. 方便用户使用，屏蔽设备的物理特性
  5. 实现与文件系统等其他模块的有机协同工作

2. I/O 控制方式

方式特点CPU 介入程度
程序询问方式CPU 不断轮询检查设备状态极高（忙等待）
中断驱动方式设备准备好后发中断通知 CPU中等
DMA 方式直接内存访问，数据不经过 CPU低（仅开始和结束）
通道方式专用 I/O 处理器执行通道程序最低
发展趋势：CPU 参与度越来越低，I/O 效率越来越高。通道是最高效的方式。

3. 缓冲技术

引入缓冲的三个目的：

  1. 改善主机和 I/O 设备之间速度不匹配的问题
  2. 提高 CPU 和 I/O 设备的并行性，提高资源利用率
  3. 减少 I/O 中断次数，放宽对 CPU 中断响应的要求

缓冲类型：单缓冲、双缓冲、循环缓冲、缓冲池

4. 磁盘数据访问

访问时间 = 寻道时间 + 旋转延迟 + 数据传输时间

  • 寻道时间：磁头移动到目标磁道的时间（通常最耗时）
  • 旋转延迟：等待目标扇区旋转到磁头下方的时间（平均为转半圈）
  • 数据传输时间：实际读写数据的时间

磁盘调度算法：

算法策略
FCFS按请求到达顺序服务
SSTF最短查找时间优先：选择离当前磁头最近的请求
SCAN（电梯算法）磁头单向移动，到头后反向
C-SCAN循环扫描：单向移动，到头后直接回到起点再扫描

第五章 文件系统

▼

1. 文件的逻辑结构

类型说明
流式文件由一串字节流组成，无结构（如 .txt, .exe）
记录式文件由若干记录组成，每条记录有特定含义

2. 文件的物理结构

结构说明优缺点
顺序结构 文件占用连续的物理块 顺序访问快，但不利于动态增长和插入删除
链接结构 文件的各物理块通过指针链接 解决了外部碎片和动态增长问题，但只能顺序访问
索引结构 为每个文件建立索引表 支持随机访问，但索引表占用额外空间

3. 文件安全

  • 访问控制：通过访问控制表 (ACL) 或用户权限列表控制
  • 备份与恢复：定期备份，建立日志
  • 访问权限：读(r)、写(w)、执行(x)
