📋 第一章 概述

四个设计目标
  • 方便性 — 方便用户使用计算机
  • 有效性 — 提高系统资源利用率
  • 可扩充性 — 适应硬件和网络发展
  • 开放性 — 遵循标准，兼容互联
四个基本特征
  • 并发性 — 多程序交替执行
  • 共享性 — 资源被多进程共用
  • 虚拟性 — 一个物理→多个逻辑
  • 不确定性 — 程序执行结果不可再现
四种操作系统结构
结构特点优缺点
整体式模块组合效率高，但模块间耦合度大
层次式分层调用结构清晰，层间接口严格
C/S（客户-服务器）请求-响应灵活分布式，但通信开销大
微内核最小内核+用户态服务安全可靠，可扩展性强
微内核的四大优势
  1. 代码量少，内核更简洁
  2. 充分模块化，更换模块不影响其他模块
  3. 未使用的模块不必运行，大幅减少内存需求
  4. 增强可移植性，修改代码量非常少
实例：华为鸿蒙 OS 采用的就是微内核结构，体现了高度模块化和可扩展性的优势。

⚙️ 第二章 处理器管理

2.1 进程

进程定义

进程是具有独立功能的程序关于某个数据集合上的一次运行活动，是系统进行资源分配、调度和保护的独立单位。

进程三态模型
就绪态
等待CPU
→ ① 调度程序选中 →
← ② 时间片用完 / 被抢占 ←
运行态
占用CPU
↓ ③ 申请资源未到 / I/O ↓
↑ ④ 资源到达 / 事件发生 ↑
等待态
等待资源/事件
不能实现的转换（老师重点强调）：
① 等待态 → 运行态（不可能）：等待事件发生后只能进入就绪队列排队，不能直接抢占 CPU。
② 就绪态 → 等待态（不可能）：就绪状态尚未拿到 CPU，不会发生需要等待的事件。

七状态模型：引入"挂起"状态，当内存不够时将进程换出到外存，用于平衡系统负载，对应中级调度。加上挂起后共有 7 种状态。
Linux 进程创建原语 fork()
  • 返回 -1：创建失败
  • 返回 0：子进程中（父进程PID）
  • 返回 子进程PID：父进程中

2.2 处理器调度 重点

三级调度体系
调度级别别名功能频率
高级调度作业调度 / 长程调度从外存选择作业进入内存最低
中级调度平衡负载 / 中程调度内存↔外存交换中等
低级调度进程调度 / 短程调度从就绪队列选进程到CPU最高
周转时间与带权周转时间
周转时间 Ti = 完成时间 Ei − 到达时间 Si
平均周转时间 T = (1/n) × Σ Ti
带权周转时间 Wi = Ti / ri（ri 为实际运行时间）
平均带权周转时间 W = (1/n) × Σ Wi
调度算法互动计算器 重点

输入任务信息，自动计算 FCFS、SJF、HRRN 三种算法的调度结果

任务名到达时间运行时间操作
教材例题详解：单道环境调度

4个任务：JOB1(8:00, 120min)、JOB2(8:50, 50min)、JOB3(9:00, 10min)、JOB4(9:50, 20min)

FCFS 先来先服务

执行顺序：JOB1→JOB2→JOB3→JOB4，平均周转时间 112.5min，平均带权周转时间 4.975

SJF 最短作业优先

JOB1完成后，从已到达的JOB2/JOB3/JOB4中选最短的。执行顺序：JOB1→JOB3→JOB4→JOB2，平均周转时间 95min，平均带权周转时间 3.25

HRRN 最高响应比优先

响应比 H = 1 + 等待时间/运行时间。10:00时计算响应比：JOB2=2.4, JOB3=7, JOB4=1.5 → 选JOB3; 10:10再算：JOB2=2.6, JOB4=2 → 选JOB2

平均周转时间 97.5min，平均带权周转时间 3.525

结论：FCFS性能最差，SJF性能最优（平均周转时间最短），HRRN介于两者之间但兼顾公平性。
计算题易错点（老师重点提醒）：
计算平均周转时间和带权周转时间时，一定要看清题目条件：是"随到随调度"还是"等所有作业全部到达后再开始调度"。两种方式结果完全不同！（参见课本75页课后题的陷阱）

调度算法练习题

▶

题目1

4个作业：A(到达0, 运行3)、B(到达1, 运行6)、C(到达3, 运行2)、D(到达5, 运行4)。分别用 FCFS 和 SJF 求平均周转时间。

FCFS：顺序 A→B→C→D
A完成=3, B完成=9, C完成=11, D完成=15
周转：A=3, B=8, C=8, D=10 → 平均 = (3+8+8+10)/4 = 7.25

SJF（1时刻后选最短）：A→C→D→B
A完成=3, C完成=5, D完成=9, B完成=15
周转：A=3, C=2, D=4, B=14 → 平均 = (3+2+4+14)/4 = 5.75

题目2

3个作业：P1(到达0, 运行24)、P2(到达0, 运行3)、P3(到达0, 运行3)。用 HRRN 求调度顺序。（提示：先 FCFS 选 P1，完成后计算响应比）

解：时刻0：三个都到达，FCFS先选P1。
P1完成时刻=24。此时计算响应比：
P2：H = 1 + 21/3 = 8；P3：H = 1 + 21/3 = 8
两者响应比相同，任选其一（如P2），再选P3。
顺序：P1→P2→P3，平均周转 = (24+26+27)/3 = 25.67

2.3 同步与互斥

PV 操作

P 操作（申请资源）

  • 信号量 S 减 1
  • 若 S < 0，进程阻塞进入等待队列

V 操作（释放资源）

  • 信号量 S 加 1
  • 若 S ≤ 0，唤醒一个等待进程
生产者-消费者问题纠错（老师课上重点分析）：
当一个进程既要申请同步信号量（如 empty/full），又要申请互斥信号量（mutex）时：
必须先 P 同步信号量，再 P 互斥信号量！
如果先 P 互斥再 P 同步，当缓冲区满/空时会造成死锁。
V 操作的顺序无所谓。
前驱图问题（PV 操作应用）

用 PV 操作控制进程执行的先后次序：

  • 前驱进程执行完后执行 V 操作（发信号）
  • 后继进程执行前执行 P 操作（等信号）
  • 每条前驱边对应一个信号量，初值为 0

PV 操作练习题

▶

题目1：读者-写者问题

多个读者可同时读，写者独占。用 PV 操作实现。

semaphore rw=1, mutex=1; int readcount=0;

读者：P(mutex) → readcount++ → 若 readcount==1 则 P(rw) → V(mutex) → 读 → P(mutex) → readcount-- → 若 readcount==0 则 V(rw) → V(mutex)

写者：P(rw) → 写 → V(rw)

题目2：前驱关系

S1→S2, S1→S3, S2→S4, S3→S4。用 PV 操作实现。

semaphore a=0, b=0, c=0;
S1; V(a); V(b);
P(a); S2; V(c);
P(b); S3; V(c); // 注意 S3 也发 c
P(c); P(c); S4; // 要等两次 c

2.4 死锁

死锁四个必要条件
互斥使用：资源独占，一次只能一个进程使用
不可抢占：资源只能由占有方自愿释放
请求和保持：申请新资源时保持已有资源
循环等待：进程间形成等待环路

死锁与预防练习题

▶

题目1（简答）

说明死锁的四个必要条件及对应的预防策略。

答：
① 互斥使用 → 一般无法破坏（资源本质属性）
② 不可抢占 → 允许系统剥夺进程已占有的资源
③ 请求与保持 → 静态分配：一次性申请所有资源
④ 循环等待 → 有序分配法：按编号递增顺序申请资源

题目2（判断）

系统有资源 A(10)、B(5)。当前分配：P1占A3+B2，P2占A3+B1。此时 Available=(4,2)。P1请求A1+B0，能否分配？

解：分配后 Available = (3,2)。
P1 还需 Max-Alloc = 需进一步确认 Max 才能判断安全性。
若仅判断资源够不够：Available(3,2) ≥ 请求(1,0)，可以尝试分配，再用银行家算法检查安全性。
三种解决策略
策略方法破坏条件
预防可抢占资源破坏"不可抢占"
预防一次性申请所有资源破坏"请求和保持"
预防资源有序分配法（按编号递增申请）破坏"循环等待"
避免银行家算法动态检查安全状态
检测+解除资源分配图检测 + 资源剥夺/撤销允许死锁发生后处理

系统有资源 (10,5,7)。5个进程的 Allocation 和 Max 如下：

AllocationMaxNeed
P0(0,1,0)(7,5,3)(7,4,3)
P1(2,0,0)(3,2,2)(1,2,2)

💾 第三章 存储管理

3.1 分页存储管理

地址转换流程
  1. 根据请求表获取进程页表起始地址和长度
  2. 读入页表控制寄存器
  3. 提取逻辑页号，与页表长度比较（越界检查）
  4. 查页表得到物理页框号
  5. 物理地址 = 页框号 + 页内地址（拼接）
引入快表（TLB）后：先查快表，命中则直接得页框号（1次访存）；未命中再查页表（2次访存）
分页地址计算方法 重点

若页面大小为 4KB = 212B，则逻辑地址的低 12 位为页内偏移，高位为页号。

例（课本101页，老师强调）：16K 页面，逻辑地址 0AC5H 和 1AC5H
页面大小 16K = 214B → 低 14 位为偏移，高位为页号。
0AC5H = 00 0010 1011 0001 01 → 页号 = 0，偏移 = 0AC5H
1AC5H = 00 0110 1011 0001 01 → 页号 = 1，偏移 = 0AC5H
查页表：页号 0 → 物理块号 X，物理地址 = X 拼接 0AC5H
注意十六进制拆分时不要算错位数！
分段地址转换与越界检查

逻辑地址 = (段号, 段内偏移)。查段表得段始址和段长。

必做步骤：越界检查！
必须先比较：段内偏移 < 段长？
• 若越界 → 产生越界中断
• 若未越界 → 物理地址 = 段始址 + 段内偏移

分页地址转换练习题

▶

题目1

页面大小 4KB，逻辑地址 0x00003A5F。页表：页号0→块号6，页号1→块号3，页号2→块号11，页号3→块号8。求物理地址。

解：4KB = 212B，低12位为页内偏移。
0x00003A5F = 0011 1010 0101 1111
页号 = 3（高4位），偏移 = 0xA5F（低12位）
查页表：页号3 → 块号8
物理地址 = 8 × 4096 + 0xA5F = 0x8A5F = 35423

题目2

页面大小 1KB，逻辑地址 2500。求页号和页内偏移。

解：1KB = 1024B
页号 = 2500 / 1024 = 2（整数部分）
偏移 = 2500 mod 1024 = 452

特性分页分段段页式

3.2 页面置换算法 重点

缺页中断率
R = F / (S + F)    （F=缺页次数, S=成功访问次数）

R 越小越好，表示大部分页面已在内存中。

三种算法对比：
  • OPT：替换将来最长时间不使用的页面（理论最优，实际无法实现）
  • FIFO：替换驻留时间最长的页面。⚠️ 可能出现 Belady 异常（增加页框数反而缺页率上升）
  • LRU：替换最近最久未访问的页面。基于局部性原理，性能接近OPT
页面置换算法互动模拟器 重点

输入页框数和访问序列，对比 OPT / FIFO / LRU 三种算法的缺页情况

页面置换练习题

▶

题目

页面引用串：7,0,1,2,0,3,0,4,2,3,0,3,2,1,2,0,1,7,0,1。分配 3 个物理块。分别用 FIFO 和 LRU 求缺页次数。

FIFO 缺页次数 = 15 次（注意可能出现 Belady 异常）
LRU 缺页次数 = 12 次

关键区别：FIFO 只看进入时间，LRU 看最近使用时间。LRU 性能通常优于 FIFO。
教材例题详解：页框数=3，访问序列 4,3,2,1,4,3,5,4,3,2,1,5

OPT 理想置换

4,3,2 入内存 → 访问1时，2最晚被再访问，替换2 → 访问4,3命中 → 访问5时，1最晚被再访问，替换1 → ...

缺页次数：7次，缺页率：7/12 ≈ 58.3%

FIFO 先进先出

4,3,2 入内存 → 访问1时，4最先进入，替换4 → 访问4时缺页（颠簸！），替换3 → ...

缺页次数：9次，缺页率：9/12 = 75%

LRU 最近最少用

4,3,2 入内存 → 访问1时，4最久未访问，替换4 → 访问4时缺页，替换3 → 访问2时，5最久未访问（与FIFO不同！）

缺页次数：10次，缺页率：10/12 ≈ 83.3%

注意 FIFO 的 Belady 异常：当页框数从3增加到4时，FIFO的缺页次数可能反而增加！这是FIFO特有的反直觉现象。用上方模拟器将页框数改为4试试。

💿 第四章 设备管理

I/O 控制方式
方式特点CPU介入
询问方式CPU不断轮询设备状态最多
中断方式设备就绪后中断通知CPU较多
DMA方式直接内存访问，以数据块为单位较少
通道方式专用I/O处理器最少
缓冲技术的目的
  • 改善CPU和I/O设备之间速度不匹配的问题
  • 提高CPU和I/O设备的并行性
  • 减少I/O中断次数
SPOOLing 技术（假脱机技术）

核心作用：利用磁盘作为缓冲，将独占设备（如打印机）改造为共享设备（虚拟设备）。

组成：输入井 / 输出井、输入缓冲区 / 输出缓冲区、输入进程 / 输出进程。
典型应用：打印机池——多个进程"同时"提交打印请求，实际由 SPOOLing 系统排队依次输出。

4.1 磁盘调度算法

磁盘数据访问时间
访问时间 = 寻道时间 + 旋转延迟 + 数据传输时间
  • 寻道时间 Ts = m × n + s（移动n条磁道）
  • 旋转延迟：平均为旋转半周的时间（如5400r/min → 平均5.55ms）
  • 数据传输：通常最短
影响性能最大的因素是寻道时间，因此优化磁盘调度算法主要目标是减少磁头移动距离。
磁盘调度算法互动模拟器

输入磁盘参数和请求队列，对比各算法的磁头移动路径

教材例题：200柱面，当前143号，请求序列 86,147,91,177,94,150,102,175,130

FCFS 先来先服务

路径：143→86→147→91→177→94→150→102→175→130

SSTF 最短查找时间优先

路径：143→147→150→130→102→94→91→86→175→177

可能产生饥饿：远端磁道请求可能被无限期推迟

电梯调度（LOOK）

路径：143→147→150→175→177→130→102→94→91→86

扫描算法（SCAN）

路径：143→147→150→175→177→199→130→102→94→91→86

单向扫描（C-SCAN）

路径：143→147→150→175→177→199→0→86→91→94→102→130

各算法对比：
  • FCFS：简单公平，但效率低
  • SSTF：效率高但可能饥饿
  • 电梯调度/SCAN：兼顾效率和公平
  • 单向扫描：适合均匀分布的请求
  • 分步扫描：解决磁臂粘着问题

📁 第五章 文件系统

逻辑结构
  • 流式文件：无结构的字节序列
  • 记录式文件：由一组记录组成
物理结构
  • 顺序结构：连续存放，访问快但有外部碎片，不利于动态增长
  • 链接结构：链表方式，无碎片易扩展，但只能顺序访问，可靠性差
  • 索引结构：通过索引表定位，支持随机访问，无碎片，但索引表占额外空间
索引文件最大长度计算 经典大题
计算套路：已知磁盘块大小（如 1KB）和盘块号占用字节数（如 4B），算出一个索引块能放的索引项数（1024/4 = 256 项）。

总文件大小 = (直接索引块数 + 一级间接索引项数 + 二级间接索引项数 + ...) × 块大小

例：13 个索引项，前 10 个直接索引，第 11 个一级间接（256 项），第 12 个二级间接（256×256 项），第 13 个三级间接。
最大文件 = (10 + 256 + 256² + 256³) × 1KB
空闲存储空间管理

位示图法（必考计算）

已知字长 \(n\)，第 \(i\) 行第 \(j\) 列：

物理块号 b = i × n + j

反之，已知块号 \(b\)：

行号 i = b ÷ n 列号 j = b mod n

成组链接法（Unix/Linux 采用）

  • 理解超级块的作用（栈顶指针）
  • 分配：取栈顶块号；若栈空则读下一组到内存
  • 回收：压入栈顶；若栈满则将当前栈写入回收块，新块作栈顶
文件安全

文件安全涉及访问控制、加密、备份恢复等机制，确保文件的机密性、完整性和可用性。

索引文件 & 位示图练习题

▶

题目1：索引文件最大长度

磁盘块大小 1KB，盘块号占 4 字节。采用三级索引结构（10个直接索引 + 1个一级间接 + 1个二级间接 + 1个三级间接）。求文件最大长度。

解：每块可放 1024/4 = 256 个索引项
直接索引：10 × 1KB = 10KB
一级间接：256 × 1KB = 256KB
二级间接：256 × 256 × 1KB = 65536KB ≈ 64MB
三级间接：256 × 256 × 256 × 1KB ≈ 16GB
最大文件 ≈ 10KB + 256KB + 64MB + 16GB ≈ 16GB+

题目2：位示图法

字长 32 位，求物理块号 b=100 对应的行号和列号；反之，第 5 行第 10 列对应哪个块号？

解：
块号 100：行 i = 100 ÷ 32 = 3，列 j = 100 mod 32 = 4
第5行第10列：b = 5 × 32 + 10 = 170

② 输出数据先暂存在磁盘的输出井中；
③ SPOOLing 输出进程依次从输出井取出数据送往打印机；
④ 对用户而言，打印机好像可以共享使用，实现了虚拟化。
