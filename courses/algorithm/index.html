第1章 算法概述

算法定义、性质、复杂度分析

▶

1.1 算法的定义与性质

▶

算法是解决问题的有规则的有限集合。

四大性质(必须理解)：
1. 有限性：执行有限步后结束
2. 确定性：每步有确切定义，无歧义
3. 可行性：现有条件下可实现
4. 输入/输出：0个或多个输入，1个或多个输出
易错点：算法可以没有输入(0个输入)，但必须有输出！

算法 vs 程序：算法具有有限性，程序可以无限运行(如操作系统)。

1.2 算法复杂性分析

▶

时间复杂度分析一般采用最坏情况下的时间复杂度。

特例：快速排序使用平均时间复杂度进行分析(实验课讲过)！

渐进复杂度排序：

O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(n³) < O(2⊃n) < O(n!) < O(n⊃n)

重要：本书中 log n 默认以 2为底，不是10也不是e！(与通信中的信噪比不同)

渐进表达式求法：

  1. 去掉所有加法项常数
  2. 只保留最高阶项
  3. 去除高阶项系数

例：3n²+10n → O(n²)，n²/100+2⊃n → O(2⊃n)，10log3⊃n = 10n·log3 → O(n)

空间复杂度：递归算法空间复杂度 = 每层辅助空间 × 递归深度

1.3 渐进分析三种表示法

▶
符号含义定义
O (低阶)f的阶不高于g∃C,n&sub0;，∀n>n&sub0;，f(n)≤Cg(n)
Ω (高阶)f的阶不低于g∃C,n&sub0;，∀n>n&sub0;，f(n)≥Cg(n)
θ (等阶)f与g同阶f=O(g) 且 f=Ω(g)

证明方法：反证法、极限法

例：证明n³不是O(n²)：假设成立则n³≤Cn²，即n≤C对所有n成立，矛盾。

第2章 递归与分治

递归概念、分治策略、排序算法

▶

2.1 递归基础

▶
递归函数两要素：
1. 递归体：递归调用部分
2. 递归出口(边界条件)：终止条件
没有递归出口 = 死循环 = 错误的递归！

经典递归：阶乘函数 n!、斐波那契数列

int factorial(int n) {
    if(n == 1) return 1;      // 递归出口
    return factorial(n-1) * n;  // 递归体
}

递归复杂度分析：替换法、递归树法、主方法

2.2 分治法基本思想

▶
分治三步骤：
1. 分解(Decompose)：大问题 → 小问题
2. 递归(Conquer)：递归求解子问题
3. 合并(Combine)：合并子问题的解
两个关键思想：
1. 平衡子问题：子问题规模大致相等(如大学分班)
2. 独立子问题：子问题之间无重叠(如一个人不能同时在两个班)
分治 vs 动态规划关键区别：分治是独立子问题，动态规划是重叠子问题！

2.3 典型分治算法

▶
算法时间复杂度要点
二分查找O(log n)有序序列，每次排除一半
归并排序O(n log n)稳定排序，需要额外空间
快速排序O(n log n) 平均用平均复杂度分析，不稳定
棋盘覆盖-了解即可
// 归并排序
void mergeSort(int a[], int left, int right) {
    if(left < right) {
        int mid = (left + right) >> 1;
        mergeSort(a, left, mid);
        mergeSort(a, mid+1, right);
        merge(a, left, mid, right);  // 合并
    }
}

2.4 主方法(Master Theorem)

▶

对形如 T(n) = aT(n/b) + f(n) 的递归方程：

情况条件结果
1f(n) = O(nlogba-ε)T(n) = θ(nlogba)
2f(n) = θ(nlogba)T(n) = θ(nlogba log n)
3f(n) = Ω(nlogba+ε)T(n) = θ(f(n))

例：T(n) = 2T(n/2) + n → a=2, b=2, f(n)=n, nlog22=n，属于情况2，T(n)=θ(n log n)

真题精选 · 算法概述 + 递归分治

▶

选择题要点

  • 算法可以有 1个或多个 输出（不是0个）
  • 操作系统程序 不是 算法（程序≠算法）
  • 动态规划通常用 数学归纳法 证明正确性
  • N皇后问题 不能用贪心算法解决
  • 分支限界法按 活结点表选择次序 分为 FIFO/LIFO/LC 三类

填空题要点

  • 回溯法 = 深度优先 + 剪枝函数；分支限界法 = 广度优先
  • FIFO 分支限界法的活结点表是 队列；LIFO 是 堆栈；LC 是 优先队列
  • 具有 最优子结构性质 是可用动态规划或贪心法求解的关键特征
  • 两路归并排序最坏时间复杂度 O(n log n)
  • 边数较少时用 Kruskal 算法效率更高

简答题：动态规划 vs 贪心法

相同点：

  • 都求解最优化问题
  • 都要求问题具有最优子结构性质

不同点：

  • 求解方式：DP 自底向上；贪心自顶向下迭代
  • 子问题依赖：DP 依赖所有子问题解；贪心不依赖子问题解，仅做局部最优选择

时间复杂度分析：

嵌套循环：for i=1..n, for j=1..i, for k=1..j → 执行次数 n(n+1)(n+2)/6 → O(n³)

while i<n: i=2*i → 执行次数 ⌊log n⌋ → O(log n)

快速排序

分治策略：选基准 x，双指针分区，递归排序左右两半。

时间：平均 O(n log n)，最坏 O(n²)

关键：基准选中间可避免最坏情况；i,j 从两端扫描。

void quick_sort(int q[], int l, int r) {
  if(l >= r) return;          // 递归出口
  int i=l-1, j=r+1;
  int x = q[l+r >>1];         // 取中间为基准
  while(i < j) {
    do i++; while(q[i] < x);  // 左找≥x
    do j--; while(q[j] > x);  // 右找≤x
    if(i < j) swap(q[i],q[j]);
  }
  quick_sort(q, l, j);         // 递归左半
  quick_sort(q, j+1, r);      // 递归右半
}

归并排序

分治策略：一分为二，递归排序后 merge 合并。

时间 O(n log n)，空间 O(n)，稳定排序。

merge：双指针取较小者放入临时数组，再拷回。

void merge(int a[],int l,int m,int r){
  int *t=(int*)malloc((r-l+1)*sizeof(int));
  int i=l, j=m+1, k=0;
  while(i<=m && j<=r)        // 双指针归并
    t[k++] = a[i]<=a[j] ? a[i++] : a[j++];
  while(i<=m) t[k++]=a[i++]; // 左剩余
  while(j<=r) t[k++]=a[j++]; // 右剩余
  for(k=0,i=l;i<=r;k++,i++) a[i]=t[k];
  free(t);
}

集合全排列

递归+回溯：依次将每个元素交换到首位，递归求剩余全排列。

时间 O(n·n!)，空间 O(n)。

注意：递归返回后必须交换回来（回溯）。

void Perm(int a[],int k,int m){
  if(k==m){                   // 递归出口：输出
    for(int i=0;i<=m;i++)
      cout<<a[i]<<" ";
  } else {
    for(int j=k;j<=m;j++){
      swap(a[k], a[j]);       // 选第j个放首位
      Perm(a, k+1, m);       // 递归剩余
      swap(a[k], a[j]);       // 回溯恢复
    }
  }
}

第3章 动态规划

最优子结构、状态转移方程、经典问题

▶

3.1 两个基本要素

▶
动态规划两大基本要素：
1. 最优子结构性质：问题最优解包含子问题最优解
2. 重叠子问题性质：子问题被反复计算
动态规划的核心：
• 状态定义：明确 dp[i] 代表什么
• 状态转移方程：当前状态与上一状态的关系(类似递归方程)
• 转移顺序：自底向上，从最小规模子问题开始
复习课重点：状态转移方程一定要会推导！给你几个矩阵/序列，要能写出方程并算出最优解。

3.2 矩阵连乘问题

▶

给定n个矩阵 {A&sub1;,A&sub2;,...,A&subn;}，其中A&subi;的维度为 p&sub(i-1); × p&subi;，求最优连乘顺序使乘法次数最少。

关键：矩阵连乘满足结合律但不满足交换律

状态定义：m[i][j] 表示 A&subi;...A&subj; 连乘的最少乘法次数

状态转移方程：

m[i][j] = {
  0                              , i = j
  min{m[i][k] + m[k+1][j]         , i ≤ k < j
      + p&sub(i-1);·p&subk;·p&subj;}
}

求解顺序：自底向上，先算1个矩阵(为0)，再算2个矩阵连乘，逐步扩大到n个

复习课强调：给若干个矩阵(如3个2×2维矩阵)，要会算最小计算次数和最优结合顺序！

3.3 最长公共子序列(LCS)

▶

子序列：不要求连续，但保持相对顺序。子串：要求连续。

状态定义：c[i][j] 表示 X&subi; 和 Y&subj; 的LCS长度

状态转移方程：

c[i][j] = {
  0                              , i=0 或 j=0
  c[i-1][j-1] + 1               , x&subi; = y&subj;
  max(c[i-1][j], c[i][j-1])      , x&subi; ≠ y&subj;
}
应用场景：版本对比(标红差异)、论文查重(主位宾结构比对)、DNA序列比对

3.4 动态规划 vs 备忘录方法

▶
对比项动态规划备忘录方法
求解方向自底向上自顶向下
实现方式迭代(填表)递归 + 缓存
子问题求解求解所有子问题只求解需要的子问题
算法复杂度相同相同
复习课提醒：AI可能说动态规划不一定是自底向上，那是因为它包含了备忘录方法。课本定义：动态规划 = 自底向上，备忘录方法是另一种形式。

3.5 0-1背包(动态规划)

▶

状态定义：dp[i][p] 表示前i个物品在背包容量为p时的最大价值

状态转移方程：

dp[i][p] = max(
  dp[i-1][p],                  // 不选第i个
  dp[i-1][p-w[i]] + v[i]       // 选第i个(需p≥w[i])
)
0-1背包用动态规划(非贪心)，分数背包用贪心算法！

第4章 贪心算法

贪心选择性质、Dijkstra、哈夫曼编码

▶

4.1 贪心算法基本要素

▶
贪心算法两大性质：
1. 贪心选择性质：通过一系列局部最优选择达到全局最优
2. 最优子结构性质：最优解包含子问题最优解(与动态规划相同)
对比相同点不同点
动态规划最优子结构重叠子问题
贪心算法最优子结构贪心选择性质
多段图最短路径：能用动态规划，不能用贪心算法！(复习课重点)
贪心不保证最优解，但通常接近最优解。在分支限界法中，贪心算法用于求下界。

4.2 Dijkstra单源最短路径

▶
复习课重点：Dijkstra迭代过程必考！不仅要知道结果，还要写出过程。

算法思想：从原点出发，每次选择距原点最近的未确定顶点加入S集合，更新distance数组。

核心概念：

  • S集合：已确定最短路径的顶点集合
  • 特殊路径：从源到u且只经过S中顶点的路径
  • 贪心选择：从V-S中选距离最小的顶点加入S

迭代过程：

  1. 初始化：S={v&sub0;}，distance[i]=cost[v&sub0;][i]
  2. 从V-S中选distance最小的顶点w加入S
  3. 更新：distance[j] = min(distance[j], distance[w]+cost[w][j])
  4. 重复直到S包含所有顶点
void Dijkstra(int cost[][], int n, int v0, int dist[], int prev[]) {
    int s[n] = {0};
    for(int i=0; i1;
    }
    dist[v0]=0; s[v0]=1;
    for(int i=1; i<n; i++) {
        // 贪心选择：找V-S中distance最小的顶点u
        int u=v0, mindis=MAX;
        for(int j=0; j<n; j++)
            if(!s[j] && dist[j]<mindis) { mindis=dist[j]; u=j; }
        s[u]=1;
        // 更新distance
        for(int j=0; j<n; j++)
            if(!s[j] && cost[u][j]<MAX)
                dist[j] = min(dist[j], dist[u]+cost[u][j]);
    }
}

4.3 哈夫曼编码

▶
核心概念 - 前缀码：任何一个字符的代码都不是其他字符代码的前缀。避免解码歧义！

应用：文本无损压缩(与图像/视频的有损压缩不同)

构建过程：

  1. 统计每个字符出现频率
  2. 每次选频率最小的两个节点合并
  3. 重复直到只剩一棵树
  4. 左分支编码0，右分支编码1

哈夫曼树特点：没有度为1的节点，权值越大的节点离根越近。

4.4 最小生成树 & 其他

▶
算法策略方法
Kruskal每次选全局最小边边排序 + 并查集判环
Prim每次扩展最近顶点从一个顶点开始扩展

分数背包(部分背包)：按单位重量价值 v/w 降序排列，依次装入。

// 分数背包 - 贪心
double greedyKnapsack(Item items[], int n, int cap) {
    sort(items, items+n, cmp);  // 按v/w降序
    double ans = 0;
    for(int i=0; i<n; i++) {
        if(cap >= items[i].w) { ans += items[i].v; cap -= items[i].w; }
        else { ans += (double)cap * items[i].v / items[i].w; break; }
    }
    return ans;
}

第4章 经典代码 · Dijkstra 双栏对照

▶

贪心策略：每次从 V-S 中选距源点最近的顶点加入 S，松弛邻居。

时间 O(n²)，可用堆优化至 O(E log V)。

关键：s[] 标记已确定最短路径的顶点，dist[] 记录当前最短距离。

void Dijkstra(int g[][],int n,int v0){
  int *s=new int[n]();       // 已确定集合
  int dist[MAXN];
  for(int i=0;i<n;i++)
    dist[i]=g[v0][i];          // 初始化
  dist[v0]=0; s[v0]=1;
  for(int i=1;i<n;i++){
    int u=0, mn=INF;
    for(int j=0;j<n;j++)    // 找最近顶点
      if(!s[j]&&dist[j]<mn)
        {mn=dist[j];u=j;}
    s[u]=1;                   // 加入S
    for(int j=0;j<n;j++)    // 松弛邻居
      if(!s[j]&&dist[u]+
         g[u][j]<dist[j])
        dist[j]=dist[u]+g[u][j];
  }
}

真题精选 · 贪心 + DP 综合（背包问题）

▶

背包问题三问（n=4, M=32, w=[6,9,10,15], p=[8,1,2,5]）

(1) 分数背包（贪心）：

按单位价值 p/w 排序：4/3, 1/9, 1/5, 1/3

贪心装入：物品0全装(6,8)→物品3全装(15,5)→物品2全装(10,2)→物品1装1/9(1,1/9)

最大收益 = 8+5+2+1/9 = 136/9

(2) 0-1 背包（DP）：

用动态规划表逐步计算，最优解 (1,0,1,1)，最大收益 = 8+2+5 = 15

(3) 图着色（回溯）：

4件物品装入3个背包，冲突图约束：相邻物品不同背包

显式约束：xᵢ ∈ {a,b,c}

隐式约束：若 (vᵢ,vⱼ) ∈ E，则 xᵢ ≠ xⱼ

所有可行解：
(a,b,c,b), (a,c,b,c),
(b,a,c,a), (b,c,a,c),
(c,a,b,a), (c,b,a,b)

第5章 回溯法

解空间树、剪枝策略、DFS

▶

5.1 基本概念

▶
回溯法 = DFS + 剪枝
目的：找出所有解
数据结构：栈(隐式递归栈)
搜索方式：深度优先搜索

解空间树类型：

类型叶子节点数典型问题
子集树2n0-1背包、子集和问题

5.2 剪枝策略

▶
两种剪枝函数：
1. 约束函数：剪去不可行解(不满足约束条件)
2. 限界函数：剪去可行但非最优解

以0-1背包为例(子集树)：

  • 左分支(选)：约束剪枝 tw+w[i] ≤ W
  • 右分支(不选)：限界剪枝，用上界函数bound判断
回溯法框架(子集树)：
void backtrack(int i) {
    if(i > n) output(x);  // 叶子节点，找到一个解
    else {
        x[i] = 1;  // 左分支：选择
        if(constraint(i) && bound(i)) backtrack(i+1);
        x[i] = 0;  // 右分支：不选
        if(constraint(i) && bound(i)) backtrack(i+1);
    }
}

N皇后问题：在N×N棋盘放N个皇后，不能同行/同列/同斜线

约束：x[i]≠x[j] 且 |i-j| ≠ |x[i]-x[j]|

int place(int j) {
    for(int i=1; i<j; i++)
        if(abs(j-i)==abs(x[j]-x[i]) || x[j]==x[i]) return 0;
    return 1;
}
void queen(int t) {
    if(t>=n) sum++;
    else for(int j=t; j<=n; j++) {
        swap(x[t], x[j]);
        if(place(t)) queen(t+1);
        swap(x[t], x[j]);
    }
}

N 皇后（排列树）

约束：不同列 x[i]≠x[j]，不同对角线 |i-j|≠|x[i]-x[j]|

框架：排列树，每行选一列，place() 检查合法性。

int place(int j) {           // 检查第j行放置是否合法
  for(int i=1;i<j;i++)
    if(abs(j-i)==abs(x[j]-x[i])
       || x[j]==x[i])         // 同列或同对角线
      return 0;
  return 1;
}
void queen(int t) {
  if(t>=n) sum++;             // 找到一个解
  else for(int j=t;j<=n;j++){
    swap(x[t], x[j]);         // 选列
    if(place(t)) queen(t+1);
    swap(x[t], x[j]);         // 回溯
  }
}

0-1 背包（子集树 + 剪枝）

左剪枝：tw + w[i] > W → 不选（约束）

右剪枝：bound(i) ≤ maxv → 不扩展（限界）

上界：按单位价值降序，贪心装入剩余物品估价值上界。

int bound(int i,int tw,int tv){
  i++;
  while(i<=n && tw+A[i].w<=W)
    {tw+=A[i].w;tv+=A[i].v;i++;}
  if(i<=n) return tv+(W-tw)*A[i].p;
  else return tv;              // 价值上界
}
void dfs(int i,int tw,int tv){
  if(i>n){/*更新最优解*/}
  else {
    if(tw+w[i]<=W){           // 左：选
      op[i]=1;
      dfs(i+1,tw+w[i],tv+v[i]);
    }
    if(bound(i,tw,tv)>maxv){  // 右：不选
      op[i]=0;
      dfs(i+1,tw,tv);
    }
  }
}

BFS搜索、限界函数、最优解

▶

6.1 基本概念

▶
分支限界法 = BFS + 剪枝
目的：找出最优解(单个)
数据结构：队列 / 优先队列(堆)
搜索方式：广度优先搜索
对比回溯法分支限界法
搜索方式DFS(深度优先)BFS(广度优先)
目的所有解最优解
数据结构栈队列/优先队列
剪枝约束+限界约束+限界
结点扩展一次一个孩子一次所有孩子

6.2 两种组织形式

▶
队列式(普通)：FIFO，按入队顺序扩展
优先队列式：按优先级(如上界值)扩展，上界越高越优先
复习课重点：优先队列分支限界法要注意使用的数据结构是堆(大根堆/小根堆)。优先级根据上界(UB)确定。

限界函数设计：每个活结点计算一个上界值，决定扩展顺序。

保存最优解向量的两种方法：

  1. 每个结点保存从根到该结点的路径(空间大但简单)
  2. 记录父节点信息，找到最优解后回溯路径(空间小)

6.3 单源最短路径(分支限界)

▶

用BFS/优先队列BFS求解：

// 优先队列式 - 用小根堆
struct Node {
    int vno, length;
    bool operator<(const Node& b) const {
        return length > b.length;  // 小根堆
    }
};
void bfs(int v) {
    priority_queue<Node> pq;
    pq.push({v, 0}); dist[v]=0;
    while(!pq.empty()) {
        Node e = pq.top(); pq.pop();
        for(int j=0; j<n; j++)
            if(a[e.vno][j]<INF && e.length+a[e.vno][j]<dist[j]) {
                dist[j] = e.length + a[e.vno][j];
                prev[j] = e.vno;
                pq.push({j, dist[j]});
            }
    }
}

第6章 经典代码 · 分支限界最短路径 双栏对照

▶

优先队列式：小根堆按 dist 值优先出队，每次扩展最近顶点。

与 Dijkstra 的区别：Dijkstra 是贪心（确定性），分支限界是 BFS+限界剪枝。

关键：Node 结构体中 operator< 实现小根堆；用 prev[] 记录路径。

struct Node {
  int vno, length;
  bool operator<(const Node&b)const{
    return length > b.length;  // 小根堆
  }
};
void bfs(int v) {
  priority_queue<Node> pq;
  pq.push({v,0}); dist[v]=0;
  while(!pq.empty()){
    Node e=pq.top(); pq.pop();
    for(int j=0;j<n;j++){
      int d=e.length+a[e.vno][j];
      if(d<dist[j]){           // 限界：更短才扩展
        dist[j]=d;
        prev[j]=e.vno;         // 记录路径
        pq.push({j, d});
      }
    }
  }
}

7.1 两种随机算法

▶
复习课：随机算法就一道2分的概念题！把概念搞清楚即可。
类型结果正确性时间确定性应用

拉斯维加斯(Las Vegas)完全正确不确定N皇后问题
记忆口诀：
蒙特卡洛 = 概率保真，可能出错(有时间保证)
拉斯维加斯 = 结果保真，可能慢(无时间保证)

应用背景(数字媒体)：

  • 图像加白噪声 → 随机算法
  • 复现实验 → 保存噪声图像 或 多组数据取平均
  • 噪声是随机的，信道突发错误是随机的
